# 前端模块技术设计

## 背景

本设计基于已确认的后端、分析推荐、参数审批与下发模块设计。前端作为一体化 SPA，负责系统总览、数据分析、参数推荐提交、审批激活、系统设置和用户操作入口。

当前设计范围覆盖前端信息架构、页面模块、关键交互流、状态管理、错误处理和测试策略，不包含后端 API 实现、Windows 安装包和设备端实现。

## 设计目标

- 提供统一 Web 入口，覆盖工艺优化闭环流程。
- 使用侧边栏后台布局，降低工艺人员和管理员学习成本。
- 前端只负责交互、渲染和调用 API，不承载核心算法。
- API 错误转化为可理解的中文提示和修复建议。
- 危险操作必须二次确认。
- 静态资源由 `BackendAPI` 托管，便于单机部署。

## 技术选型

- 框架：Vue 3。
- UI 组件：Element Plus。
- 图表：ECharts。
- 路由：Vue Router。
- 状态管理：Pinia。
- 构建工具：Vite。
- HTTP 客户端：Axios 或基于 `fetch` 的统一封装。

选择 Vue 3 + Element Plus 的原因：

- 适合后台管理系统。
- 表单、表格、对话框、菜单组件成熟。
- 中文生态和企业内部系统实践丰富。
- 构建结果可作为静态资源由 FastAPI 托管。

## 总体布局

采用侧边栏后台布局。

区域：

- 顶部栏：系统名称、当前用户、系统状态、登出入口。
- 侧边栏：主导航菜单。
- 内容区：当前页面主体。
- 全局提示区：错误、成功、告警消息。

主导航：

- 系统总览。
- 数据分析。
- 工艺管理。
- 系统设置。
- 用户中心。

系统设置仅管理员可见。未登录用户只能访问登录页。

## 路由设计

路由按业务模块拆分。

### 系统总览

- `/dashboard`

用途：展示系统健康状态和关键指标。

### 数据分析

- `/analysis`
- `/analysis/correlation`
- `/analysis/regression`
- `/analysis/recommendation`

用途：筛选数据、运行分析、生成推荐参数。

### 工艺管理

- `/parameters`
- `/parameters/:id`
- `/parameters/approvals`
- `/parameters/confirmations`

用途：管理参数集、审批、激活和查看设备确认。

### 系统设置

- `/settings`
- `/settings/services`
- `/settings/config`
- `/settings/backup`
- `/settings/logs`

用途：服务状态、配置、备份恢复和日志查看。

### 用户中心

- `/account`
- `/login`

用途：登录、登出和查看当前用户信息。

## 页面模块设计

### 系统总览

展示内容：

- 服务运行状态：Gateway、Consumer、BackendAPI、NATS、PostgreSQL。
- 今日上报数据量。
- 最近数据更新时间。
- 接入设备数。
- 待审批参数集数量。
- 最近错误或告警。
- CPU、内存、磁盘占用。

交互：

- 点击服务状态进入系统设置详情。
- 点击待审批数量进入审批列表。
- 点击数据量进入数据分析页面。

### 数据分析

页面结构：

- 筛选区：时间范围、设备、条码、批次、字段选择。
- 数据概览区：样本数、缺失率、字段摘要。
- 分析标签页：相关性、特征重要性、回归、推荐。
- 结果区：图表、表格、解释文本。
- 操作区：生成推荐、提交为建议参数集。

交互流程：

1. 用户选择时间范围、设备和字段。
2. 前端调用数据概览接口。
3. 用户选择分析类型。
4. 前端调用对应分析 API。
5. 展示图表、指标和解释。
6. 用户进入推荐页，填写可调参数和约束。
7. 前端展示推荐结果和风险提示。
8. 用户提交为 `proposed` 参数集。

前端不执行核心算法，只展示后端返回的结果。

### 工艺管理

页面结构：

- 参数集列表：名称、设备类型、版本、状态、来源、创建人、更新时间。
- 参数集详情：参数项、版本、状态、审批历史、设备确认。
- 审批列表：待审批参数集。
- 激活操作区：激活已审批参数集。
- 设备确认列表：设备拉取、应用成功、失败状态。

交互：

- `draft` 可编辑和提交。
- `proposed` 可审批通过或驳回。
- `approved` 可激活。
- `active` 可查看设备确认状态。
- `approved`、`active`、`archived` 不允许直接编辑，只能复制为新版本。

激活操作必须二次确认，并提示旧 active 参数集会自动归档。

### 系统设置

仅管理员可见。

页面模块：

- 服务控制：查看、启动、停止、重启服务。
- 配置管理：查看和编辑数据库、NATS、API Key 等配置。
- 备份恢复：创建数据库备份、恢复备份。
- 日志查看：查看 Gateway、Consumer、BackendAPI 日志。

危险操作：

- 重启全部服务。
- 停止服务。
- 恢复数据库备份。
- 修改连接配置。

危险操作必须二次确认，恢复备份需明确提示会影响当前数据。

### 用户中心

首版支持简单能力：

- 登录。
- 登出。
- 查看当前用户。
- 查看角色权限。

不实现复杂组织架构和细粒度 RBAC。

## 状态管理

使用 Pinia 分模块管理状态。

### session store

保存：

- 当前用户。
- 角色。
- 登录状态。
- Token 或会话信息。

### permission store

保存：

- 可访问路由。
- 可见菜单。
- 可执行操作。

### system store

保存：

- 服务健康状态。
- 系统资源状态。
- 最近告警。

### analysis store

保存：

- 当前筛选条件。
- 字段选择。
- 最近一次分析结果。
- 推荐结果草稿。

### parameters store

保存：

- 参数集列表筛选条件。
- 当前参数集详情。
- 审批列表。
- 设备确认数据。

## API Client 设计

前端通过统一 API Client 访问后端。

职责：

- 附加认证信息。
- 统一处理 `401`，跳转登录。
- 统一处理结构化业务错误。
- 显示 loading 状态。
- 对只读请求支持可选重试。
- 对写操作不自动重试，避免重复提交。

错误展示：

- `EMPTY_DATASET`：没有符合条件的数据，请调整筛选条件。
- `INSUFFICIENT_SAMPLES`：样本量不足，请扩大时间范围或减少字段。
- `FIELD_NOT_FOUND`：字段不存在，请重新选择字段。
- `NON_NUMERIC_FIELD`：字段不是数值类型，不能参与分析。
- `MODEL_FIT_FAILED`：模型拟合失败，请调整字段或样本范围。
- `INVALID_TRANSITION`：当前状态不允许执行该操作。
- `ACTIVE_CONFLICT`：参数激活冲突，请刷新后重试。
- `NO_ACTIVE_PARAMETER_SET`：当前设备类型暂无生效参数。

## 权限与可见性

首版角色：

- 工艺人员：访问总览、数据分析、创建和提交参数集。
- QA/主管：访问总览、工艺管理、审批参数集。
- 管理员：访问全部模块，执行系统设置和激活操作。
- 设备 API Key：不进入前端，仅访问设备 API。

前端权限用于交互约束，后端仍必须执行最终权限校验。

## 关键交互流

### 分析到推荐闭环

1. 用户进入数据分析。
2. 选择时间、设备、工艺参数和检测指标。
3. 查看相关性和回归结果。
4. 配置推荐目标、可调参数和约束。
5. 生成推荐参数。
6. 查看预测值、风险提示和备选方案。
7. 提交为 `proposed` 参数集。

### 审批到下发闭环

1. QA/主管进入审批列表。
2. 查看参数集详情、来源、推荐解释和参数项。
3. 审批通过或驳回。
4. 管理员激活 approved 参数集。
5. 设备通过 API 主动拉取 latest 参数。
6. 前端查看设备确认状态。

### 运维闭环

1. 管理员进入系统设置。
2. 查看服务状态和日志。
3. 对异常服务执行重启。
4. 必要时创建备份或恢复备份。
5. 返回系统总览确认状态恢复。

## 图表设计

数据分析页使用 ECharts 展示：

- 相关性热力图。
- 特征重要性柱状图。
- 回归预测值与实际值散点图。
- 残差图。
- 推荐参数对比表。

图表需要支持：

- loading 状态。
- 空状态。
- 错误状态。
- 下载图片或导出数据后续扩展。

## 空状态与异常状态

前端必须明确处理：

- 首次进入页面无筛选条件。
- 筛选结果为空。
- 样本不足。
- 后端服务不可用。
- 用户无权限。
- 长时间分析超时。
- 参数集状态已被他人更新。

每种状态都应给出下一步建议。

## 测试策略

### 路由与权限测试

- 未登录访问业务页跳转登录。
- 普通用户不可见系统设置。
- 无权限操作按钮隐藏或禁用。

### API Client 测试

- `401` 跳转登录。
- 结构化错误转换为中文提示。
- 只读请求可重试。
- 写操作不自动重试。

### 数据分析页面测试

- 筛选表单校验。
- 分析结果图表渲染。
- 样本不足错误展示修复建议。
- 推荐结果提交为 `proposed`。

### 工艺管理页面测试

- 参数集列表按状态筛选。
- 审批通过和驳回操作。
- 激活操作二次确认。
- 设备确认状态展示。

### 系统设置页面测试

- 危险操作二次确认。
- 服务状态展示。
- 日志列表展示。
- 备份恢复入口权限限制。

## 非目标

本阶段不实现：

- 复杂拖拽式 BI。
- 前端算法计算。
- 前端直连数据库或 NATS。
- 复杂组织架构管理。
- 多租户前端。
- 移动端专项适配。
